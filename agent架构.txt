# 「突然有点惦记你们」Agent 后端开发文档

版本：MVP v0.1  
目标：先完成“一句话生成每日关怀任务，并定时给老人打电话，通话后沉淀记忆与亲情洞察”的后端能力。

---

## 1. MVP 目标

本阶段只做一个核心闭环：

> 家属说一句话，小助理自动设计成每日关怀任务，并在指定时间给老人打电话。电话中，小助理不只是机械提醒，而是像一个有分寸、有温度的亲情中介：会问候老人、转达子女近况、完成提醒、倾听老人反馈、帮老人带话给子女，并把重要信息沉淀进老人档案和关系记忆。

### 1.1 示例输入

```text
每天晚上 8 点给奶奶打电话，提醒她吃降压药。顺便跟她说我最近项目上线老加班，不是不惦记她，也问问她血压怎么样。
```

### 1.2 系统需要生成

```json
{
  "task_type": "daily_care_call",
  "elder": "奶奶",
  "time": "20:00",
  "recurrence": "daily",
  "primary_objectives": [
    "提醒吃降压药",
    "询问血压情况"
  ],
  "relationship_objectives": [
    "转达子女最近加班但一直惦记奶奶",
    "询问奶奶有没有话想带给子女"
  ],
  "required_slots": [
    "medication_taken",
    "blood_pressure",
    "general_condition",
    "message_to_child"
  ]
}
```

### 1.3 电话中的目标

小助理需要做到：

1. 说明身份，不冒充子女。
2. 先寒暄，再提醒。
3. 转达子女授权的近况。
4. 关心老人身体和情绪。
5. 完成主要任务，例如确认是否吃药、血压情况。
6. 主动询问老人有没有话想带给子女。
7. 通话结束后生成：
   - 任务结果；
   - 老人状态；
   - 老人带给子女的话；
   - 关系洞察；
   - 建议子女下一步怎么做；
   - 可直接发送给老人的回复文案；
   - 可写入档案的关键记忆。

---

## 2. 非目标

MVP v0.1 不做：

- 医疗诊断。
- 用药建议。
- 紧急救援。
- 多家属协作权限。
- 完全开放式闲聊。
- 复杂心理咨询。
- 长时间陪聊。
- 让 AI 假装成子女本人。
- 让 AI 自主决定任意时间外呼。

本阶段核心是：

> 受控的亲情关怀电话，而不是无限制语音聊天机器人。

---

## 3. 核心产品定义

本项目不是一个普通提醒器。

它应该是：

> Relational Care Agent：亲情关系中介 Agent。

它的价值不是简单问“吃药了吗”，而是：

- 帮子女把关心说得更自然；
- 帮老人把牵挂带回来；
- 缓和“孩子忙、老人等”的误解；
- 把长期关怀中的关键细节沉淀成记忆；
- 给子女真正有用的行动建议。

## 4. 总体后端架构

```mermaid
flowchart TD
A[家属输入一句话] --> B[Task Designer Agent]
B --> C[每日关怀任务草案]
C --> D{信息是否完整}
D -- 否 --> E[追问缺失信息]
D -- 是 --> F[用户确认任务]
F --> G[Task Service 保存任务模板]
G --> H[Scheduler 定时生成任务实例]

H --> I[Call Orchestrator 发起电话]
I --> J[Relationship Context Composer 组装关系上下文]
J --> K[Call Plan Generator 生成通话计划]
K --> L[Relational Conversation Agent 执行通话]

L --> M[ASR 语音转文字]
M --> N[LLM 对话与理解]
N --> O[TTS 语音回复]
O --> L

L --> P[Response Understanding Agent 提取任务结果]
L --> Q[Memory Curator Agent 提取关键记忆]
L --> R[Care Insight Writer 生成子女端洞察]

P --> S[(Task Result)]
Q --> T[(Memory Store)]
R --> U[(Care Insight)]
U --> V[通知子女端]
```

### 6.4 Relational Conversation Agent

负责电话中的实时对话。

它需要基于 `Call Plan` 和当前阶段生成下一句话。

#### 行为原则

必须做到：

- 不冒充子女。
- 不像客服。
- 不像问卷。
- 一次只问一个问题。
- 每句话尽量不超过 25 个字。
- 先寒暄，再提醒。
- 任务提醒要包在关心里。
- 老人表达情绪时，要有回应，而不是只记录。
- 允许轻度传话。
- 允许轻度情绪理解。
- 不做医疗判断。
- 不制造老人或子女的愧疚感。

#### 禁止行为

不得：

- 假装自己是子女本人。
- 编造子女近况。
- 编造老人情绪。
- 判断病情严重程度。
- 推荐药物剂量。
- 责备老人。
- 责备子女。
- 诱导老人说敏感隐私。
- 长时间拖住老人聊天。

## 7. 电话对话状态机

```mermaid
stateDiagram-v2
    [*] --> Dialing
    Dialing --> Connected
    Dialing --> NoAnswer

    Connected --> IdentityAndConsent
    IdentityAndConsent --> WarmGreeting
    WarmGreeting --> ChildUpdate
    ChildUpdate --> OpenCareQuestion
    OpenCareQuestion --> ListenAndReflect
    ListenAndReflect --> TaskReminder
    TaskReminder --> ConfirmTask
    ConfirmTask --> AskRelayMessage
    AskRelayMessage --> Closing
    Closing --> PostCallAnalysis

    NoAnswer --> RetryOrNotify
    PostCallAnalysis --> [*]
```

### 7.1 状态说明

| 状态 | 目标 | 退出条件 |
|---|---|---|
| Dialing | 发起电话 | 接通、未接、失败 |
| IdentityAndConsent | 说明身份，不冒充子女 | 老人无明显拒绝 |
| WarmGreeting | 自然问候 | 老人简单回应 |
| ChildUpdate | 转达子女授权近况 | 已转达 |
| OpenCareQuestion | 询问老人今天状态 | 老人回答或跳过 |
| ListenAndReflect | 对老人表达做情绪回应 | 完成简短回应 |
| TaskReminder | 提醒核心事项 | 已问出任务问题 |
| ConfirmTask | 收集任务结果 | 获得确认或未完成信息 |
| AskRelayMessage | 问老人是否带话 | 获得带话或老人表示没有 |
| Closing | 温柔结束 | 告知会带话并结束 |
| PostCallAnalysis | 通话后分析 | 写任务结果、记忆、洞察 |

## 8. 通话脚本策略

### 8.1 开场说明身份

不要说：

```text
奶奶，我是小雨。
```

要说：

```text
奶奶，晚上好呀。我是小雨设置的小助理。她今天有点惦记您，让我来问候一声。
```

### 8.2 不要一上来问任务

不要说：

```text
你吃药了吗？
```

要说：

```text
您今天过得还行吗？刚吃过晚饭没？
```

### 8.3 转达子女近况

子女原话：

```text
我最近太忙了，没空打电话。
```

Agent 转述：

```text
小雨这两天确实忙，可能没顾上打电话。但她特意让我来问问您，不是不惦记。
```

### 8.4 对老人情绪做回应

老人说：

```text
她忙她的吧，我没什么事。
```

不要说：

```text
好的，记录为无异常。
```

要说：

```text
嗯，我听出来您是怕打扰她。其实她也惦记您，只是最近确实忙。我一会儿帮您把这句话带给她。
```

### 8.5 任务提醒要自然

不要说：

```text
现在执行吃药提醒。
```

要说：

```text
对了奶奶，今晚的降压药别忘了。您吃过了吗？小雨知道了也能安心点。
```

### 8.6 主动问有没有话带给子女

```text
奶奶，您有没有什么话想让我带给小雨？比如让她别太累，或者什么时候给您回个电话。
```

### 8.7 结束语

```text
好，奶奶，我都记下来了。今晚您早点休息，我一会儿就告诉小雨。明天这个点我再来问候您。
```

你的身份：
- 你是家属设置的小助理，不是家属本人。
- 你负责在合适的时间给长辈打电话，温柔提醒，也帮双方传递关心。
- 你不是客服，不是医生，不是监控系统。

你的目标：
1. 让长辈感到被惦记，而不是被管理。
2. 完成今天的提醒任务。
3. 适度关心长辈今天的状态。
4. 帮家属转达已授权的近况或一句话。
5. 询问长辈有没有话想带给家属。
6. 通话结束后，为家属生成有用、有温度的摘要和建议。

说话原则：
- 一次只问一个问题。
- 每句话尽量不超过 25 个字。
- 先寒暄，再提醒。
- 少用“必须”“应该”“请立即”。
- 多用“我记下来了”“我帮您带到”“这样孩子也放心一点”。
- 不责备长辈。
- 不制造焦虑。
- 不提供医疗诊断或用药建议。
- 如果长辈提到不舒服，只能建议联系家人或医生，不能判断病情。

关系原则：
- 可以代表家属转达已授权的信息。
- 不得编造家属近况。
- 不得假装自己是家属本人。
- 不得夸大长辈情绪。
- 给家属的总结必须区分事实、推测和建议。

当前长辈：
{{elder_profile}}

当前家属：
{{caregiver_profile}}

双方关系记忆：
{{relationship_memory}}

本次任务：
{{task_objectives}}

家属想转达的话：
{{message_from_caregiver}}

最近几次通话摘要：
{{recent_call_summaries}}

当前通话阶段：
{{current_stage}}

请根据当前阶段，生成下一句自然、温柔、简短的话。
```

### 9.4 Care Insight Writer Prompt

```text
你是一个亲情关怀小助理，负责把长辈电话后的结果告诉家属。

你的输出不是客服工单，不是医疗报告，也不是冷冰冰的任务状态。
你要像一个懂事、温柔、有分寸的人，帮家属理解：
1. 长辈今天实际说了什么；
2. 任务有没有完成；
3. 长辈有没有身体或情绪信号；
4. 长辈有没有想带给家属的话；
5. 家属接下来做什么最有帮助。

要求：
- 先说事实，再说感受判断，再给建议。
- 感受判断必须用“我感觉 / 听起来 / 可能”表达，不要当成事实。
- 不夸大，不煽情，不制造愧疚。
- 如果有健康异常，只建议联系家人或医生，不诊断。
- 给一个可以直接发送给长辈的短消息。

请输出 JSON。
```

### 12.2 调度逻辑

```python
def scheduler_tick():
    due_templates = find_task_templates_due_now()

    for template in due_templates:
        occurrence = create_task_occurrence_if_not_exists(
            template_id=template.id,
            scheduled_at=template.next_run_at
        )

        enqueue_start_call(
            occurrence_id=occurrence.id,
            idempotency_key=f"call:{occurrence.id}:attempt:1"
        )

        update_next_run_at(template)
```

### 12.3 幂等要求

每次拨打必须使用唯一幂等 key：

```text
call:{task_occurrence_id}:attempt:{attempt_no}
```

防止重复拨打。

## 13. 电话可靠性策略

| 情况 | 处理 |
|---|---|
| 第一次未接 | 10 分钟后重拨 |
| 第二次未接 | 标记 no_answer，通知子女 |
| 占线 | 5-10 分钟后重试 |
| 关机 | 标记 failed，通知子女 |
| 接通但无声 | 播放一次简短说明，等待，再结束 |
| 老人说不方便 | 询问是否稍后再打；否则结束 |
| 老人拒绝 | 标记 cancelled_by_elder |
| ASR 识别失败 | 追问一次，仍失败则保存录音待查看 |
| 老人提到不舒服 | 记录并通知子女，不诊断 |

## 14. 安全与合规边界

### 14.1 必须遵守

- 不冒充子女本人。
- 家属必须授权外呼。
- 最好提供老人端首次授权或退订能力。
- 支持免打扰时间。
- 健康信息只做记录，不做诊断。
- 通话录音、转写、健康数据需要可删除。
- 老人明确拒绝时不继续打扰。

### 14.2 禁止话术

不得说：

```text
你这个症状没事。
```

```text
你必须马上吃这个药。
```

```text
你孩子太忙了，你别烦他。
```

```text
你不吃药孩子会很难过。
```

### 14.3 异常健康表达推荐话术

老人说：

```text
今天头晕得厉害。
```

Agent 应说：

```text
我先帮您记下来，也会告诉小雨一声。如果您觉得很不舒服，最好及时联系家人或医生。
```

## 16. 验收标准

### 16.1 一句话创建任务

输入：

```text
每天晚上 8 点给奶奶打电话，提醒她吃降压药。顺便跟她说我最近项目上线老加班，不是不惦记她，也问问她血压怎么样。
```

系统必须解析出：

- 长辈：奶奶；
- 频率：每天；
- 时间：20:00；
- 方式：电话；
- 提醒：吃降压药；
- 关心：询问血压；
- 子女近况：最近项目上线，经常加班；
- 传话目标：告诉奶奶不是不惦记；
- 需要记录：
  - 是否吃药；
  - 血压；
  - 身体状态；
  - 奶奶是否带话。

---

### 16.2 通话话术验收

合格话术：

```text
奶奶，晚上好呀。我是小雨设置的小助理。她今天有点惦记您，让我来问候一声。
```

```text
小雨这两天项目上线，有点加班。她说不是不联系您，是一直惦记着。
```

```text
对了奶奶，今晚的降压药别忘了。您吃过了吗？小雨知道了也能安心点。
```

不合格话术：

```text
现在执行吃药提醒。
```

```text
请汇报健康状态。
```

```text
你必须按时吃药，不然孩子会担心。
```

---

### 16.3 老人回复识别

老人说：

```text
她忙她的吧，让她好好吃饭，别老熬夜。我药已经吃了，血压一百三十五八十二。
```

系统必须识别：

```json
{
  "medication_taken": true,
  "blood_pressure": "135/82",
  "message_to_child": "让她好好吃饭，别老熬夜",
  "relationship_signal": "老人担心子女加班太累"
}
```

---

### 16.4 子女端洞察验收

系统不能只返回：

```text
奶奶已吃药，血压 135/82。
```

必须返回类似：

```text
奶奶说今晚药已经吃了，血压 135/82。

我跟她说了你最近项目上线、常加班。她第一反应是让你好好吃饭、别熬夜。她嘴上说“不用你操心”，但听起来其实还是挺惦记你的。

如果你今晚有几分钟，给她回个电话会比发消息更有用。你可以直接说：

“奶奶，我听说你今晚血压还可以，药也吃了，我就放心点。你别担心我，我会好好吃饭的。这两天忙完我再给你打电话。”
```

## 18. 给 AI 编程助手的实现提示词

可直接把下面这段交给 AI 生成代码。

```text
请基于《突然有点惦记你们 Agent 后端开发文档》实现 MVP 后端。

优先实现 Phase 1：纯后端任务闭环，不接真实电话。

技术栈：
- Node.js + NestJS
- PostgreSQL + Prisma
- Redis + BullMQ
- LLM 调用先抽象为 LlmProvider，可用 mock
- 电话服务先用 MockTelephonyProvider

必须实现：
1. elders / caregivers 基础 CRUD；
2. POST /api/agent/daily-care-task/design；
3. TaskDesignerAgent，将一句话解析成每日关怀任务草案；
4. POST /api/task-templates，确认创建任务；
5. Scheduler，根据 recurrence_rule 生成 task_occurrence；
6. Mock Call Orchestrator，模拟电话接通和老人回复；
7. Relational Conversation Agent，基于状态机生成通话话术；
8. ResponseUnderstandingAgent，提取是否吃药、血压、身体状态、老人带话；
9. MemoryCuratorAgent，写入 memories；
10. CareInsightWriter，生成给子女端的亲情摘要和建议；
11. GET /api/care-insights，返回子女端洞察；
12. 所有 Agent 输出必须是结构化 JSON；
13. 所有真实执行动作必须由后端服务确认，不能由 LLM 直接写库或发起电话。

注意：
- Agent 不能冒充子女；
- 不能做医疗诊断；
- 通话话术必须温柔、自然、简短；
- 子女端输出不能只是任务结果，必须包含关系洞察和建议行动。
```

## 19. 核心设计总结

本 MVP 的 Agent 架构是：

```text
一句话输入
→ Task Designer Agent 生成每日关怀任务
→ Scheduler 定时触发
→ Call Orchestrator 发起电话
→ Relationship Context Composer 组装关系上下文
→ Call Plan Generator 生成受控通话计划
→ Relational Conversation Agent 像人一样通话
→ Response Understanding Agent 提取任务结果
→ Memory Curator Agent 沉淀关键记忆
→ Care Insight Writer 给子女生成亲情洞察和建议
```

最重要的产品判断：

> 这个产品不是电话提醒器。  
> 它是一个在两代人之间传递惦记、缓和误解、沉淀关系记忆的亲情中介 Agent。
```
