你的方案方向是对的，**适合快速做 Hackathon/MVP Demo**，但我建议你做几处关键修改：

1. **不要叫“真实语音电话”**：你现在用的是浏览器 TTS/STT，本质是“浏览器内语音通话模拟”，还不是拨打真实手机号的电话。
2. **需要增加后端 Call Session 状态机**：否则只是前端语音交互，无法沉淀记忆、生成亲情洞察。
3. **TTS/STT 要防止互相打架**：AI 播放时不能开麦，否则浏览器可能把 AI 自己的声音识别成老人回复。
4. **STT 不要完全依赖 continuous: true**：`SpeechRecognition` 兼容性有限，并且在部分浏览器中不是稳定能力；MDN 也标注 SpeechRecognition 不是 Baseline，且部分浏览器会把音频发送到服务端识别。([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/SpeechRecognition))
5. **需要增加“通话计划 + 对话阶段”**：不是收到一句老人回复就让 AI 自由发挥，而是按阶段推进：问候 → 转达子女近况 → 关怀 → 提醒 → 带话 → 结束。
6. **结束后必须触发 post-call analysis**：提取任务结果、老人状态、带话、关系信号、关键记忆、给子女的建议。

下面是我建议你直接替换后的版本。

## 6. 前端轮次控制逻辑

核心原则：

> AI 说话时，STT 必须关闭。  
> AI 说完后，才开启麦克风听老人说话。

否则会出现：

- AI 的 TTS 被 STT 识别成老人说话；
- 对话轮次混乱；
- 重复触发后端；
- 字幕和状态错乱。

### 6.1 推荐轮次流程

```text
1. 后端返回 assistant_text
2. 前端进入 ai_speaking
3. TTS 播放 assistant_text
4. TTS onEnd
5. 如果 is_call_ending=true，进入 ending/ended
6. 否则进入 elder_listening
7. 开启 STT
8. 用户说话
9. 检测 final transcript 或静音超时
10. 停止 STT
11. 进入 processing
12. transcript 发送到 /api/calls/{sessionId}/turn
13. 后端返回下一句 assistant_text
14. 循环
```

## 7. Task 1：创建 TTS Hook

文件：

```text
app/components/hooks/useSpeechSynthesis.ts
```

### 7.1 功能要求

封装：

```ts
window.speechSynthesis
SpeechSynthesisUtterance
```

Web Speech API 的 SpeechSynthesis 负责文本转语音；可通过 `speechSynthesis.speak()` 播放 `SpeechSynthesisUtterance`。([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/Web_Speech_API))

### 7.2 必须提供

```ts
type SpeakOptions = {
  text: string;
  lang?: string;
  rate?: number;
  pitch?: number;
  volume?: number;
  onStart?: () => void;
  onEnd?: () => void;
  onError?: (error: unknown) => void;
};

type UseSpeechSynthesisReturn = {
  supported: boolean;
  speaking: boolean;
  voices: SpeechSynthesisVoice[];
  selectedVoice?: SpeechSynthesisVoice;
  speak: (options: SpeakOptions) => void;
  stop: () => void;
  pause: () => void;
  resume: () => void;
};
```

### 7.3 中文语音选择规则

优先级：

1. `zh-CN`
2. `zh-Hans`
3. `zh`
4. 浏览器默认 voice

注意：

- `getVoices()` 返回的是当前设备可用 voices 列表。([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/SpeechSynthesis/getVoices))
- voices 可能异步加载，需要监听 `voiceschanged` 事件更新 voice 列表。([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/SpeechSynthesis/voiceschanged_event))

### 7.4 TTS 播放策略

每次 speak 前必须：

```ts
window.speechSynthesis.cancel();
```

避免上一次语音残留排队。

### 7.5 推荐默认参数

```ts
{
  lang: "zh-CN",
  rate: 0.92,
  pitch: 1.0,
  volume: 1.0
}
```

老人场景下：

- 语速略慢；
- 音量默认最大；
- 不要太机械；
- 句子短，减少长段 TTS 失败风险。

## 8. Task 2：创建 STT Hook

文件：

```text
app/components/hooks/useSpeechRecognition.ts
```

### 8.1 功能要求

封装：

```ts
window.SpeechRecognition || window.webkitSpeechRecognition
```

SpeechRecognition 是 Web Speech API 的语音识别控制接口；MDN 标注该特性并非 Baseline，因为它不适用于一些主流浏览器。([developer.mozilla.org](https://developer.mozilla.org/en-US/docs/Web/API/SpeechRecognition))

### 8.2 必须提供

```ts
type UseSpeechRecognitionReturn = {
  supported: boolean;
  listening: boolean;
  interimTranscript: string;
  finalTranscript: string;
  error?: string;
  startListening: () => void;
  stopListening: () => void;
  resetTranscript: () => void;
};
```

### 8.3 配置建议

```ts
recognition.lang = "zh-CN";
recognition.continuous = false;
recognition.interimResults = true;
recognition.maxAlternatives = 1;
```

### 8.4 为什么不建议默认 continuous: true

原方案写的是：

```ts
continuous: true
```

建议改成：

```ts
continuous: false
```

原因：

- 对话是轮次制，不需要一直监听；
- 一直监听容易吃进 AI TTS；
- 浏览器可能自动停止识别；
- 静音、权限、设备切换时更难控制；
- 对老人电话场景，明确的“一轮听一句”更稳定。

如果确实需要连续识别，可以后续作为配置项开启。

### 8.5 必须处理的事件

```ts
recognition.onstart
recognition.onresult
recognition.onerror
recognition.onend
recognition.onspeechstart
recognition.onspeechend
recognition.onaudioend
recognition.onnomatch
```

### 8.6 静音超时

需要加一个 silence timeout。

推荐策略：

```text
用户开始说话后，如果 1.2-1.8 秒没有新的 final/interim 内容，则认为本轮结束。
```

或者提供按钮：

```text
我说完了
```

作为 fallback。

### 8.7 错误处理

必须处理：

| 错误 | 前端提示 |
|---|---|
| not-allowed | 麦克风权限被拒绝，请允许浏览器使用麦克风 |
| no-speech | 没听清，可以再说一遍 |
| audio-capture | 没有检测到麦克风 |
| network | 语音识别服务暂时不可用 |
| aborted | 本轮识别已取消 |
| unsupported | 当前浏览器不支持语音识别 |

## 12. 前端核心伪代码

```ts
async function handleAssistantText(text: string, isCallEnding: boolean) {
  setCallState("ai_speaking");

  speechRecognition.stopListening();

  speechSynthesis.speak({
    text,
    onEnd: () => {
      if (isCallEnding) {
        endCall();
      } else {
        startElderListening();
      }
    },
    onError: () => {
      setCallState("error");
    }
  });
}

function startElderListening() {
  setCallState("elder_listening");
  speechRecognition.resetTranscript();
  speechRecognition.startListening();
}

async function submitElderTurn(text: string) {
  setCallState("processing");
  speechRecognition.stopListening();

  const res = await fetch(`/api/calls/${callSessionId}/turn`, {
    method: "POST",
    body: JSON.stringify({
      elderText: text
    })
  }).then(res => res.json());

  await handleAssistantText(res.assistantText, res.isCallEnding);
}
```

“奶奶，我听说你今晚血压还可以，药也吃了，我就放心点。你别担心我，我会好好吃饭的。这两天忙完我再给你打电话。”

## 16. 必须增加的降级方案

由于浏览器 SpeechRecognition 兼容性有限，需要提供 fallback。

### 16.1 浏览器不支持 STT

展示：

```text
当前浏览器不支持语音识别，你可以用文字模拟长辈回复。
```

提供：

```text
文字输入框
```

### 16.2 麦克风权限被拒绝

展示：

```text
麦克风权限被拒绝。请允许浏览器使用麦克风，或改用文字输入。
```

### 16.3 TTS 不可用

展示：

```text
当前浏览器无法播放语音，我会改用文字展示小助理的话。
```

### 16.4 STT 识别不准

提供按钮：

```text
重新说一遍
我说完了
改用文字输入
重播上一句
```

## 17. 验证与调试

### 17.1 基础验证

- 页面可以打开 VoiceCallModal；
- dialing 状态正常；
- connected 后 AI 自动说第一句话；
- AI 说话时 STT 未开启；
- AI 说完后 STT 自动开启；
- 用户说话时字幕实时显示；
- 用户说完后 transcript 发送给后端；
- 后端返回下一句 assistantText；
- AI 继续说话；
- `isCallEnding=true` 时通话结束；
- ended 页面展示摘要。

### 17.2 兼容性验证

至少测试：

- Chrome desktop；
- Edge desktop；
- Safari desktop；
- Chrome Android；
- Safari iOS。

其中 Safari/iOS 的 STT 兼容性需要重点验证。

### 17.3 异常验证

需要测试：

- 用户拒绝麦克风权限；
- 浏览器不支持 SpeechRecognition；
- TTS 播放失败；
- 网络请求失败；
- 后端超时；
- 用户一直不说话；
- 用户中途挂断；
- AI 正在说话时用户点击挂断；
- 后端返回 isCallEnding。

## 18. 本版本验收标准

### 18.1 通话体验

必须完成：

```text
拨号 → 接通 → AI 语音 → 用户语音 → AI 回应 → 结束 → 摘要
```

### 18.2 轮次控制

必须保证：

- AI 说话时不开麦；
- 用户说话时不播放 TTS；
- 一轮用户输入只发送一次后端；
- 后端返回后才进入下一轮。

### 18.3 Agent 人设

AI 必须：

- 说明自己是小助理；
- 不冒充子女；
- 语言温柔自然；
- 不像客服；
- 不做医疗诊断；
- 不责备老人；
- 能帮子女转达近况；
- 能问老人是否有话带给子女。

### 18.4 通话后结果

必须生成：

- 任务结果；
- 老人状态；
- 老人带话；
- 关系洞察；
- 建议子女行动；
- 可写入记忆的内容。

## 20. 总结

当前方案建议从：

```text
真实语音电话功能
```

调整为：

```text
浏览器语音电话 MVP
```

本阶段重点不是“真的拨出电话”，而是先验证：

1. AI 能不能自然说话；
2. 用户能不能语音回复；
3. 前端能不能顺畅轮次切换；
4. 后端能不能推进亲情对话状态机；
5. 通话后能不能沉淀记忆和生成子女端洞察。

真正的手机号外呼，应放到 Phase 2 接电话服务商实现。
```

---

一句话总结：  
**你的方案可以保留，但要把它定位成“浏览器语音通话 MVP”，并补上后端 Call Session、对话状态机、Post-call Analysis、降级方案和 TTS/STT 轮次互斥控制。**否则它会更像一个前端语音 Demo，而不是「突然有点惦记你们」真正需要的亲情中介 Agent。
